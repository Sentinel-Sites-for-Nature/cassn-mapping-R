# 2026-03 SSN Deployment

Maps UCNRS sentinel site deployment progress across 40 reserves.

## What this project does

- Joins authoritative reserve coordinates with a user-maintained status table
- Produces a static map showing each reserve colored by deployment stage

## How to update

1. Edit `data_raw/ucnrs_deployment_status.csv` — update the `status` or `notes` columns for any reserve
2. Re-run `scripts/01_process.R` to regenerate the GeoPackage
3. Re-render `scripts/02_map.qmd` to regenerate the map

`reserve_name` in `ucnrs_deployment_status.csv` must match `ucnrs_reserves.csv` exactly. The processing script warns on any mismatches.

## Data

### Inputs

| File | Description |
|------|-------------|
| `data_raw/ucnrs_reserves.csv` | Authoritative reserve coordinates — columns: `reserve_name`, `campus`, `lat`, `lon` |
| `data_raw/ucnrs_deployment_status.csv` | User-maintained status file — columns: `reserve_name`, `status`, `notes` |

### Output (generated)

`data/ucnrs_deployment.gpkg` — spatial point layer joining both CSVs. Do not edit manually; regenerated by `01_process.R`.

## Status levels

| Status | Color |
|--------|-------|
| Not yet visited | `#D9D9D9` (light gray) |
| Infrastructure set up | `#F2C300` (yellow) |
| Collecting data (1st round) | `#74C476` (light green) |
| Collecting data (2nd round) | `#238B45` (dark green) |

## Scripts

| Script | Role |
|--------|------|
| `00_setup.R` | Install and load required packages into project-local `.Rlib` |
| `01_process.R` | Read CSVs → create spatial points → join status → write GeoPackage |
| `02_map.qmd` | Render static PNG map with legend and site-count table |

## Outputs

`outputs/maps/ucnrs_deployment_progress.png` — 11×8.5" at 300 dpi.

## Typical run order

1. Edit `data_raw/ucnrs_deployment_status.csv` with current status
2. Run `scripts/00_setup.R`
3. Run `scripts/01_process.R`
4. Render `scripts/02_map.qmd`:
   ```
   quarto render scripts/02_map.qmd
   ```
